---
title: Local Data and Backups
description: How MikroCanvas stores local boards.
---

MikroCanvas stores boards in browser IndexedDB when it runs in local mode. Data stays in the current browser profile unless you export it or the browser profile syncs storage through its own features.

When online boards are enabled, boards are saved in the MikroCanvas API SQLite database instead. The browser keeps a small local list of board IDs it has opened so the board library can show recently used online boards without exposing a public board directory.

For online boards created in the current browser, MikroCanvas also stores a local delete token. That token is not part of copied board links, so clearing browser storage may remove the browser's ability to delete that board unless an admin token is configured on the server.

## Storage availability

If IndexedDB is unavailable, MikroCanvas keeps the current board in memory for the session and shows the board surface so work can continue. Export JSON before leaving the page in that situation.

## Backups

Use JSON export for portable backups. A JSON board file can be shared as an attachment, stored with project files, or kept in normal file storage.

For API-backed boards, back up the SQLite database file configured with `databasePath`.

## Browser data clearing

Clearing site data, browser storage, or the browser profile can remove local boards. Export important boards before clearing data.
